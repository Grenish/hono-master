# Contributing to Hono Master

Thanks for your interest in contributing! Hono Master is a community skill, and improvements, bug fixes, and new reference guides are welcome.

## How to Contribute

1. **Fork the repository** and create a feature branch from `main`.
2. **Make your changes** — whether it's fixing a typo, improving a pattern, or adding a new reference guide.
3. **Follow the existing style** — keep code examples consistent with the TypeScript + Bun style used throughout.
4. **Test your examples** — ensure any code snippets are correct and runnable.
5. **Open a pull request** with a clear title and description of what you changed and why.

## What to Contribute

- Corrections or improvements to existing reference guides
- New reference guides (e.g., caching, background jobs, feature flags)
- Better patterns or best practices for Hono + Bun
- Fixes to code examples or configuration snippets

## Guidelines

- Keep examples concise and focused on a single concept.
- Prefer Hono's built-in middleware over third-party packages unless there's a strong reason.
- Use `bun:test` for testing examples and `app.request()` for HTTP-level tests.
- Reference the official [Hono documentation](https://hono.dev) when appropriate.

## Code of Conduct

All contributors must adhere to our [Code of Conduct](CODE_OF_CONDUCT.md).
